# Modpack Manager

A desktop tool for administering and publishing a Minecraft modpack hosted on GitHub.

## Features

- **Full recursive scan** of the `pack/` directory — mods, config, kubejs, shaderpacks, resourcepacks, and any future folder.
- **SHA-256 hashing** (streaming, memory-efficient) computed in parallel via `ThreadPoolExecutor`.
- **Manifest generation** (`manifest.json`) with file path, name, size, hash, modification time, type, and relative download URL.
- **Change detection** — added, modified, deleted, and renamed (hash-based) files.
- **Semantic versioning** (`version.json`) — patch, minor, major increments or manual entry.
- **Changelog generation** (`CHANGELOG.md`) with optional admin notes.
- **Export** — `manifest.json`, `CHANGELOG.md`, and a plain-text summary.
- **`.manifestignore`** support (gitignore-style patterns).
- **Modern GUI** built with CustomTkinter (dark/light/system theme).
- **Persistent config** (`config.json`) — remembers last project, last increment type, window size, theme.

## Requirements

- Python 3.11+
- [customtkinter](https://github.com/TomSchimansky/CustomTkinter)

```
pip install -r tools/requirements.txt
```

## Running

```
cd modpack/tools
python main.py
```

## Workflow

1. **Select project folder** — point to the `modpack/` directory.
2. **Bump version** — click Patch / Minor / Major, or type a version manually.
3. **Scan project** — hashes all files in `pack/`, computes diff vs. previous manifest.
4. **Review the summary** in the log panel.
5. **Generate Manifest** — writes `manifest.json`.
6. **Generate Changelog** — prepends a new block to `CHANGELOG.md` (optional notes prompt).
7. **`git commit` + `git push`** — done manually; the tool only prepares the local files.

## Architecture notes

The codebase is intentionally layered so each concern lives in exactly one module:

| Module | Responsibility |
|---|---|
| `config.py` | Persistent JSON config |
| `logger.py` | Thread-safe log dispatcher |
| `utils.py` | Ignore filters, formatting helpers |
| `hashing.py` | SHA-256 streaming, parallel worker |
| `versioning.py` | `version.json` read / write / bump |
| `scanner.py` | Directory walk → `FileEntry` list |
| `manifest.py` | Manifest build, diff, save, export |
| `changelog.py` | `CHANGELOG.md` append, export |
| `gui.py` | CustomTkinter window, wires everything |
| `main.py` | Entry point |

Future extensions (multiple modpacks, branches, cloud storage providers) can be plugged in by implementing new provider/strategy classes without touching the existing modules.
